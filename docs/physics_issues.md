# Physics System - Проблемы и решения

## Известные проблемы (исправлено)

### 1. Grounded flag нестабилен — исправлено

**Проблема**: При стоянии на земле или платформе `grounded` периодически сбрасывается в `False`, хотя объект находится на поверхности.

**Причина**: 
- В методе `update()` физического тела `grounded` сбрасывается в `False` в начале каждого кадра
- Затем в `_resolve_collisions()` устанавливается в `True` при коллизии
- Из-за дискретизации позиции (через `int()`) объект может "отделиться" от поверхности на 1 пиксель между кадрами

**Реализовано**:
- **Near-ground**: после разрешения коллизий вызывается `_apply_near_ground()` — если низ спрайта в пределах `NEAR_GROUND_PX` (3 px) от верха статики и есть горизонтальное перекрытие, выставляется `grounded = True` и позиция подтягивается к поверхности. Устраняет джиттер при стоянии.
- По-прежнему используется `_grounded_cooldown` для корректного сброса grounded после прыжка.

### 2. Прохождение сквозь объекты на высокой скорости — исправлено

**Проблема**: При высокой скорости объект может проскочить сквозь тонкую стену/платформу.

**Причина**: Дискретная проверка коллизий — проверяется только позиция после перемещения.

**Реализовано**: В `PhysicsWorld` добавлен **substepping** (по умолчанию 4 подшага за кадр). За один кадр выполняется `substeps` раз: `body.update(dt/substeps)` и `_resolve_collisions()`. Настройка: `PhysicsWorld(gravity=980, substeps=4)` (можно увеличить до 6–8 для очень быстрых объектов).

## Реализованные функции

### BodyType (Enum)
- `DYNAMIC` - полная физика (игроки, мячи)
- `STATIC` - неподвижные (стены, пол)
- `KINEMATIC` - управляемые движение (движущиеся платформы) - в разработке

### PhysicsConfig
- `mass` - масса тела
- `gravity` - гравитация
- `friction` - трение
- `bounce` - отскок
- `body_type` - тип тела

### PhysicsBody
- `apply_force(force)` - приложить силу
- `apply_impulse(impulse)` - мгновенный импульс
- `set_velocity(x, y)` - установить скорость
- `set_bounce(bounce)` - отскок
- `set_friction(friction)` - трение
- `stop()` - остановить
- `grounded` - флаг на земле

### PhysicsWorld
- `add(body)` - добавить тело
- `add_static(body)` - добавить статическое
- `add_kinematic(body)` - добавить кинематическое
- `remove(body)` - удалить
- `set_bounds(rect)` - границы мира

## Что было протестировано

### Тест 1: Падение на пол
```
Frame 0-15: grounded=False (падение)
Frame 16+: grounded=True (на земле)
```
Работает стабильно.

### Тест 2: Отскок от пола
```
Frame 16: grounded=True (удар о пол)
Frame 17-25: grounded=False (отскок вверх)
```
После отскока grounded сбрасывается - это корректное поведение.

### Тест 3: Стояние на месте
```
Frame 16+: grounded=True стабильно
```
После введения near-ground проверки джиттер устранён.
