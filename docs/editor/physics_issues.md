# Physics System — примечания (pymunk)

С версии 2.2.0 физика SpritePro реализована на **pymunk**. Ниже — примечания по поведению и ограничениям.

---

## Реализация

- **Движок**: pymunk (Chipmunk2D). Единицы: пиксели, секунды; ось Y вниз положительная.
- **Формы**: задаются через **PhysicsShape** (AUTO, BOX, CIRCLE, LINE) или строки; по умолчанию — из спрайта (прямоугольник по `sprite.rect`, круг при `sprite_shape == "circle"`, линия при `sprite_shape == "line"`).
- **Синхронизация**: при включённой физике **только позиция** спрайта управляется телом (dynamic/kinematic); угол поворота **не синхронизируется** (из тела в спрайт не записывается, из спрайта в тело тоже не передаётся). Это предотвращает рост `rect` при повороте. У static тело каждый кадр следует за позицией спрайта. Любое изменение позиции спрайта (set_position, position, rect) сразу телепортирует привязанное тело.

---

## Grounded

Для динамических тел флаг **grounded** вычисляется после каждого шага физики (segment query вниз от центра тела). Используется для условия прыжка. Возможны кратковременные сбросы на 1 кадр при быстром движении — при необходимости можно добавить небольшой cooldown в игровой логике.

---

## Границы мира (set_bounds)

Границы обрабатываются **после** `space.step(dt)`: позиция динамических тел зажимается внутри rect, скорость по соответствующей оси отражается с учётом bounce. Статические segment'ы по границе не создаются.

---

## Коллизии (pymunk 6 vs 7)

Код поддерживает и pymunk 6, и pymunk 7: при отсутствии метода `add_default_collision_handler` используется `space.on_collision(None, None, post_solve=...)` для вызова `on_collision`.

---

## Туннелирование

При очень высокой скорости объект может проскакивать сквозь тонкие стены. В pymunk можно увеличить частоту шага (меньше `dt` при вызове `update(dt)`) или использовать более толстые коллайдеры. Substepping внутри одного кадра при необходимости настраивается в `PhysicsWorld(substeps=N)`.
