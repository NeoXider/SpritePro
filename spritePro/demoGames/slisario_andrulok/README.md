# Слизарио

Мультиплеерный клон [Slither.io](https://slither.io/) на **SpritePro**: змейка за курсором, еда, боты, камера follow и host-authoritative синхронизация по TCP. Демо задумано как **эталон сетевой игры на SpritePro** — как делить владение объектами между хостом и клиентами, см. [ARCHITECTURE.md](ARCHITECTURE.md).

![Слизарио — скриншот игры](../../../assets/slizario.png)

## Об авторе

Демо-игра **andrulok** — учебный/пет-проект в составе демо SpritePro (папка `slisario_andrulok`).

Геймплей «змейки в стиле Slither.io» без игрового движка: движение головы за курсором, цепочка сегментов по трейлу, сбор еды и рост, столкновения со стенами и чужими телами, боты с простым ИИ, HUD со счётом, пингом и таблицей лидеров. Для мультиплеера используется встроенный networking SpritePro (`s.run(multiplayer=True)`, `MultiplayerContext`, `send_every` / `poll`).

## Запуск

```bash
pip install spritepro

# Одиночная игра (из корня репозитория)
python spritePro/demoGames/slisario_andrulok/main.py --single

# Локально: хост + клиент, два окна
python spritePro/demoGames/slisario_andrulok/main.py --quick

# Хост (сервер + первое окно)
python spritePro/demoGames/slisario_andrulok/main.py --host_mode --host 127.0.0.1 --port 5050

# Клиент (нужен уже запущенный сервер; можно подключаться в середине игры)
python spritePro/demoGames/slisario_andrulok/main.py --host 127.0.0.1 --port 5050
```

Управление: **курсор** — направление; после смерти — **R** (рестарт, в мультиплеере — рespawn в случайной точке).

## Правила

- Еда даёт +1 к счёту; каждые 3 еды — +1 сегмент.
- Врезался головой в стену, чужое тело или чужую голову — погиб; тело превращается в еду.
- Боты гибнут о тела игроков и друг друга и возрождаются через 3 секунды.

## Структура

- `game/config.py` — все константы, включая сетевые интервалы
- `game/snake.py` — змейка локального игрока (движение, рост, состояние для сети)
- `game/bot.py` — ИИ ботов (только на хосте / в одиночной игре)
- `game/food.py` — еда с id, снапшоты и сверка на клиенте
- `game/net_view.py` — отображение чужих змей без симуляции
- `game/world.py` — фон и стены
- `scenes/game_scene.py` — геймплей, столкновения, весь сетевой обмен
- `ui/hud.py` — счёт, пинг, таблица лидеров, экран смерти
- `ARCHITECTURE.md` — почему сеть устроена именно так
