# Урок 4. Лобби и управление списком игроков

В этом уроке мы разберем, как отслеживать подключение новых игроков и синхронизировать общий список участников (Roster).

## 1. Идентификаторы игроков (Client ID)

В SpritePro каждому участнику присваивается уникальный `client_id` (целое число):
- **Хост (Сервер + Игрок 1)**: всегда имеет `id = 0`.
- **Клиенты**: получают `id > 0` от сервера при подключении.

Вы можете узнать свой ID через `ctx.client_id` только после того, как флаг `ctx.id_assigned` станет `True`.

## 2. Событие входа (Join)

Когда новый игрок подключается, он должен заявить о себе. Обычно это делается отправкой пустого события `"join"`.

**Важно:** В SpritePro сетевой слой автоматически добавляет поле `sender_id` в каждое сообщение. Вам не нужно вручную прописывать «кто отправил», сервер увидит это сам.

```python
# Клиент: отправляем сигнал о входе
if ctx.id_assigned and not joined:
    ctx.send("join")
    joined = True
```

## 3. Синхронизация списка (Roster)

Клиенты не знают друг о друге напрямую — они знают только сервер. Поэтому именно Хост должен собирать список всех ID и рассылать его остальным.

**Логика на Хосте:**
```python
player_ids = set([0]) # Хост всегда в списке

for msg in ctx.poll():
    if msg.get("event") == "join":
        pid = msg.get("sender_id")
        player_ids.add(pid)
        
        # Рассылаем обновленный список всем
        ctx.send("roster", {"players": list(player_ids)})
```

**Логика на Клиенте:**
```python
for msg in ctx.poll():
    if msg.get("event") == "roster":
        # Обновляем локальный список имен/ID
        active_players = msg.get("data")["players"]
```
