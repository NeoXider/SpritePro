# Урок 5. UI и сцены: лобби и переход в игру

В больших сетевых проектах важно отделять меню от игрового процесса. В этом уроке мы научимся переключать сцены синхронно для всех игроков.

## 1. Зачем нужны Сцены в мультиплеере?

Сцена (`Scene`) — это изолированный блок логики и графики. 
- **MenuScene**: отрисовывает кнопки, поле ввода имени и список игроков. Здесь работает логика «готовности».
- **GameScene**: отрисовывает карту, персонажей и обрабатывает физику.

Разделение позволяет обнулять состояние игры (удалять старые спрайты), не затрагивая сетевое подключение.

## 2. Использование компонентов UI

SpritePro предоставляет готовые компоненты, такие как `Button`. Они автоматически обрабатывают наведение и нажатие.

**Пример кнопки готовности:**
```python
def toggle_ready():
    self.is_ready = not self.is_ready
    ctx.send("ready", {"value": self.is_ready})

button = s.Button(
    pos=(400, 300),
    text="Ready",
    on_click=toggle_ready,
    scene=self # Кнопка привязывается к текущей сцене
)
```

## 3. Синхронное переключение

Главная ошибка — переключать сцену только у того, кто нажал кнопку. В сетевой игре это должен сделать **Хост**, отправив команду остальным.

**Логика перехода:**
```python
# На Хосте (в MenuScene.update)
if all_ready:
    ctx.send("start", {}) # Команда всем клиентам
    s.scene.set_scene_by_name("game") # Переход у себя

# На Клиенте (в MenuScene.update)
for msg in ctx.poll():
    if msg.get("event") == "start":
        s.scene.set_scene_by_name("game") # Переход по команде
```

## Практика

1. Изучите `example_menu_scenes.py`. Обратите внимание, как устроены классы `MenuScene` и `GameScene`.
2. Запустите пример (`s.run(multiplayer=True, ...)`). 
3. Нажмите кнопку `Ready` в обоих окнах. Хост автоматически переключит всех в игровую сцену, где можно двигаться.
