# Урок 5. UI и сцены: меню и старт игры

## Цель
- Разделить меню и игру на сцены.
- Использовать UI‑кнопки для готовности.
- Переключать сцены по сетевому событию `start`.

## Требования
- Python 3.7+
- spritePro доступен в проекте

## Теория (кратко)
- Сцены позволяют разделять логику меню и игры.
- UI в меню не должен мешать логике игры.
- Сетевые события удобны для синхронного старта.

## EventBus и прямые сообщения
- EventBus — для игровых событий (shoot, ready, start), удобно иметь подписчиков.
- Прямые `net.send()`/`send_every()` — для частых пакетов (позиции), меньше оверхеда.
- EventBus не заменяет сетевой слой, это просто удобная надстройка.

## Практика
1. Запустите `example_menu_scenes.py`.
1. Нажмите кнопку Ready в каждом окне.
1. Убедитесь, что сцена меняется после старта.
