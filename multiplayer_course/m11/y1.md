# Урок 11: Сетевые Декораторы и Контекст Лобби (Mirror-style API)

В этом уроке мы научимся использовать мощную абстракцию — **сетевые декораторы**, которые делают написание сетевого кода в SpritePro невероятно похожим на работу с **Mirror API** в Unity.
Декораторы позволяют нам избавиться от длинных конструкций `if event == "..."` и прозрачно рассылать или принимать сообщения вызовом одной функции!

Кроме того, мы разберем, как получить детальную информацию об игроках, подключенных в лобби, используя `multiplayer_ctx`.

---

## 1. Что такое сетевые декораторы?

В библиотеке `spritePro.net_decorators` доступны 3 основных декоратора:

1. `@Command` — Функция вызывается на любом клиенте или хосте, но **физически выполняется только на хосте**. Это идеально для логики, где только сервер (хост) должен принимать решения (например, нанести урон, заспавнить объект).
2. `@ClientRpc` — Функция вызывается хостом, но **выполняется на всех клиентах (и самом хосте)**. Подходит для визуальных эффектов или оповещений, которые сервер отправляет обратно всем (например, звук взрыва, победа игрока).
3. `@NetEvent(event_name)` — Простая обертка. Кто угодно может вызвать функцию, и она будет **перенаправлена всем** по сети. Идеально для общего чата или простого пинга.

### Важное правило!
Чтобы авто-вызов функций работал, в вашем главном цикле `update` **обязательно** должна происходить выборка сообщений из сети. 
 SpritePro в режиме лобби (через `s.run()`) автоматически делает `ctx.poll()`, поэтому декораторы работают прозрачно и "из коробки"!

---

## 2. Использование декораторов на примере

Рассмотрим кусок кода из нашей демо-сцены:

```python
from spritePro.net_decorators import Command, ClientRpc, NetEvent
import spritePro as s

class DecoratorDemo(s.Scene):
    def __init__(self):
        super().__init__()
        self.ctx = s.multiplayer_ctx

    # Хост будет вычислять изменения позиции. Кто угодно может "попросить" хоста сдвинуть объект.
    @Command
    def request_move(self, data: dict):
        # Этот код выполняется ТОЛЬКО НА ХОСТЕ
        new_x = data["x"] + 10
        # Хост решил, что позиция изменилась, и уведомляет всех
        self.apply_move({"x": new_x})

    # Хост оповещает всех об изменении
    @ClientRpc
    def apply_move(self, data: dict):
        # Этот код выполняется у всех
        print(f"Новая позиция для объекта: {data['x']}")

    # Просто событие, без жестких ограничений (шлют все, получают все)
    @NetEvent("global_ping")
    def on_ping(self, data: dict):
        print(f"Пинг от игрока {data['sender']}")

    def update(self, dt):
        # Если мы хотим сдвинуть объект:
        if s.input.is_key_just_pressed(s.pygame.K_SPACE):
            # Мы просто вызываем функцию! 
            # Декоратор сам запакует её в сетевой пакет и отправит на хост.
            self.request_move({"x": 100})
```

Вам больше не нужно делать вручную:
```python
ctx.send("request_move", {"x": 100})
# ... и в другом месте ...
for msg in ctx.poll():
    if msg.get("event") == "request_move":
        # и так далее
```
Всё это берет на себя декоратор!

## 3. Мощь `multiplayer_ctx` и получение списка игроков лобби

Если вы пользуетесь встроенной сценой лобби `use_lobby=True`, то все клиенты перед входом в игру вводят свой **никнейм**, получают **Client ID**, и выбирают статус "Готов".
Как только игра началась, лобби всё ещё живет "в фоне", и мы в любой момент можем запросить информацию обо всех членах группы, чтобы отобразить их ники над головой или в UI.

Делается это с помощью специального метода `get_players()` у вашего контекста `ctx`:

```python
def print_all_players():
    ctx = s.multiplayer_ctx
    if not ctx:
        return

    # Это вернет словарь, где ключи - client_id, а значения - словари с информацией
    players_dict = ctx.get_players()
    
    # Пример формата players_dict:
    # {
    #    0: {"name": "Ivan_Server", "role": "host", "ready": True, "ip": "127.0.0.1"},
    #    1: {"name": "Petr", "role": "client", "ready": True, "ip": "192.168.0.15"}
    # }

    for cid, info in players_dict.items():
        name = info.get("name", f"Player {cid}")
        is_host = (info.get("role") == "host")
        print(f"Игрок: {name} | ID: {cid} | ホст: {is_host}")
```

### Как это использовать на практике?
При появлении нового игрового персонажа (создании его спрайта) вы просто запрашиваете из лобби его `name` по `client_id` (например, отправленному в сетевом пакете позиции) и устанавливаете текст на UI или `TextSprite` прямо над его моделькой.
