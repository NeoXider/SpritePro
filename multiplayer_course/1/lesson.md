# Урок 1. Сетевой слой SpritePro: сообщения и очередь

В этом уроке мы разберем фундамент сетевого взаимодействия в SpritePro: как данные превращаются в пакеты и как работает очередь сообщений.

## 1. Формат сообщения: Event/Data

SpritePro использует JSON-протокол, где каждое сообщение — это объект с двумя обязательными полями:
- **`event`** (string): Тип события (например, `"move"`, `"chat"`, `"hit"`).
- **`data`** (any): Любые данные, связанные с событием (числа, строки, словари).

**Пример структуры пакета:**
```json
{
  "event": "damage",
  "data": { "hp": 80, "source": "player_1" }
}
```

## 2. Механизм Poll (Опрос очереди)

Сеть работает асинхронно. Когда приходят данные, они попадают во внутренний буфер (очередь). Чтобы обработать эти данные в игровом цикле, нужно вызвать метод `.poll()`.

> [!IMPORTANT]
> Метод `.poll()` возвращает список сообщений, накопленных с момента предыдущего вызова. Если не вызывать его каждый кадр, очередь переполнится, а задержка (пинг) будет расти.

**Пример обработки на клиенте:**
```python
# В игровом цикле (update)
for msg in net.poll():
    event = msg.get("event")
    data = msg.get("data")
    
    if event == "welcome":
        print(f"Сервер сказал: {data['text']}")
```

## 3. Сервер и Клиент: Роли

### Сервер (`NetServer`)
- Слушает входящие подключения.
- Принимает сообщения от всех клиентов.
- Может делать **Broadcast** (рассылку) всем игрокам.

```python
server = s.NetServer(host="0.0.0.0", port=5050)
server.start()

# Рассылка всем
server.broadcast("game_start", {"time": 3})
```

### Клиент (`NetClient`)
- Подключается к серверу по IP и порту.
- Отправляет сообщения только на сервер.

```python
client = s.NetClient("127.0.0.1", 5050)
client.connect()

# Отправка на сервер
client.send("hello", {"name": "Ivan"})
```
