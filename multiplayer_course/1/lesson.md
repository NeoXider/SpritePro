# Урок 1. Сетевой слой SpritePro: сообщения и очередь

## Цель

Понять, как SpritePro передаёт данные между процессами: формат JSON-сообщений, очередь входящих пакетов (poll), роли сервера и клиента.

## 1. Формат сообщения: Event + Data

SpritePro использует JSON-протокол. Каждое сообщение — объект с двумя полями:

| Поле | Тип | Описание |
|------|-----|----------|
| `event` | `str` | Тип события: `"move"`, `"chat"`, `"hit"`, `"ready"` |
| `data` | `dict` / `list` / `int` / ... | Полезная нагрузка события |

Пример пакета, который приходит из сети:
```json
{
  "event": "damage",
  "data": { "hp": 80, "source": "player_1" }
}
```

**Что можно класть в `data`?** Всё, что сериализуется в JSON:
- Числа: `42`, `3.14`
- Строки: `"hello"`
- Списки: `[100, 200]` (для позиций)
- Словари: `{"hp": 80, "name": "Ivan"}`

> **Нельзя** использовать: `Vector2`, `pygame.Rect`, сложные Python-объекты. Преобразуйте их в списки/числа перед отправкой.

## 2. Сервер (`NetServer`)

Сервер слушает входящие подключения и управляет всеми клиентами.

```python
import spritePro as s

# Создаём сервер на порту 5050.
# host="0.0.0.0" означает: слушать на всех сетевых интерфейсах.
server = s.NetServer(host="0.0.0.0", port=5050, debug=False)
server.start()  # Запускает слушающий поток (неблокирующий)
```

**Основные методы сервера:**

| Метод | Что делает |
|-------|-----------|
| `server.start()` | Запускает приём подключений в отдельном потоке |
| `server.poll()` | Возвращает `list[dict]` — все накопленные входящие сообщения |
| `server.broadcast(event, data)` | Рассылает сообщение **всем** подключённым клиентам |
| `server.stop()` | Останавливает сервер и закрывает все соединения |

Пример обработки входящих:
```python
for msg in server.poll():
    event = msg.get("event")       # Строка: "hello", "move" и т.д.
    data = msg.get("data")         # Словарь с данными
    sender = msg.get("sender_id")  # ID клиента-отправителя (int)
    
    if event == "hello":
        name = data.get("name", "unknown")
        print(f"Клиент {sender} представился: {name}")
        # Отвечаем broadcast — все клиенты получат welcome
        server.broadcast("welcome", {"text": f"Привет, {name}!"})
```

## 3. Клиент (`NetClient`)

Клиент подключается к серверу по IP и порту.

```python
# 127.0.0.1 — loopback (та же машина). Для игры в сети — IP сервера.
client = s.NetClient("127.0.0.1", 5050)
client.connect()  # Устанавливает TCP-соединение (блокирующий)
```

**Основные методы клиента:**

| Метод | Что делает |
|-------|-----------|
| `client.connect()` | Устанавливает TCP-соединение с сервером |
| `client.send(event, data)` | Отправляет сообщение на сервер |
| `client.poll()` | Возвращает `list[dict]` — все входящие от сервера |
| `client.disconnect()` | Закрывает соединение |

Пример отправки и приёма:
```python
# Отправляем событие hello на сервер
client.send("hello", {"name": "Ivan"})

# Через некоторое время читаем ответ
for msg in client.poll():
    if msg.get("event") == "welcome":
        text = msg["data"]["text"]
        print(f"Сервер ответил: {text}")
```

## 4. Механизм Poll (Опрос очереди)

Сеть работает **асинхронно**: данные приходят в любой момент и складываются во внутренний буфер. Метод `.poll()` забирает **все** накопленные сообщения разом.

```
Сеть (фон)     Буфер (очередь)     Ваш код (update)
   ↓                ↓                    ↓
  msg1 ──→ [msg1]
  msg2 ──→ [msg1, msg2]
  msg3 ──→ [msg1, msg2, msg3]
                                     poll() → [msg1, msg2, msg3]
                    []               (буфер очищен)
```

> **Правило:** вызывайте `.poll()` **каждый кадр** (или каждую итерацию цикла). Если не вызывать — буфер растёт, задержка увеличивается.

## 5. Полный пример: loopback-тест

Один процесс запускает и сервер, и клиента — удобно для отладки:

```python
"""Loopback: сервер + клиент в одном процессе."""
import time
import spritePro as s


def main():
    # 1. Запускаем сервер
    server = s.NetServer(host="0.0.0.0", port=5050, debug=False)
    server.start()

    # 2. Подключаем клиента к тому же порту
    client = s.NetClient("127.0.0.1", 5050)
    client.connect()

    # 3. Клиент отправляет hello
    client.send("hello", {"name": "player"})

    # 4. Крутим цикл 5 секунд
    start = time.time()
    while time.time() - start < 5.0:
        # Сервер: обрабатываем входящие
        for msg in server.poll():
            if msg.get("event") == "hello":
                name = msg["data"]["name"]
                print(f"[server] Получен hello от {name}")
                server.broadcast("welcome", {"text": f"Привет, {name}!"})

        # Клиент: обрабатываем ответы
        for msg in client.poll():
            if msg.get("event") == "welcome":
                print(f"[client] {msg['data']['text']}")

        time.sleep(0.05)  # 20 итераций/сек (не нагружаем CPU)

    # 5. Завершение
    client.disconnect()
    server.stop()
    print("Готово!")


if __name__ == "__main__":
    main()
```

**Что произойдёт при запуске:**
```
[server] Получен hello от player
[client] Привет, player!
Готово!
```
