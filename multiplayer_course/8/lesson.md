# Урок 8. Финальная сборка и структура проекта

Когда игра перерастает простой скрипт, важно организовать код так, чтобы в нем не запутаться. В этом уроке мы научимся выносить настройки в отдельный файл и структурировать классы.

## 1. Конфигурация игры (`game_config.py`)

Не храните магические числа (скорость, лимит очков, таймеры) прямо в коде сцен. Вынесите их в отдельный файл. Это позволит менять баланс игры, не заходя в основной код.

**Пример `game_config.py`:**
```python
# Настройки баланса
TARGET_SCORE = 10
MOVE_SPEED = 300.0
SYNC_INTERVAL = 0.05 # 20 раз в секунду

# Настройки графики
SCREEN_SIZE = (1024, 768)
COLOR_RED = (220, 70, 70)
COLOR_BLUE = (70, 120, 220)
```

## 2. Разделение ответственности

Ваша игра должна состоять из четких уровней:
1. **Сетевой уровень**: Обработка `ctx.poll()` и отправка пакетов. Лучше всего инкапсулировать это в методы сцен или отдельный менеджер.
2. **Игровой уровень**: Движение спрайтов, проверка столкновений.
3. **UI уровень**: Отображение текста и кнопок.

## 3. Правильный запуск проекта

В SpritePro 3.x рекомендуется использовать единую точку входа.

```python
import spritePro as s
import game_config as cfg

def main():
    # Запуск с поддержкой мультиплеера
    s.run(
        multiplayer=True,
        multiplayer_entry=game_start,
        size=cfg.SCREEN_SIZE
    )

def game_start():
    # Инициализация сцен и старт цикла
    s.scene.add_scene("game", MyGameScene)
    s.run(setup=lambda: s.scene.set_scene_by_name("game"))
```
