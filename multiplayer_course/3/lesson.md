# Урок 3. Состояние готовности и старт игры

В этом уроке мы создадим базовую логику лобби: игроки должны подтвердить готовность (`Ready`), прежде чем Хост запустит игру.

## 1. Синхронизация логических флагов

В сетевой игре важно, чтобы все участники знали статус друг друга. Мы будем использовать словарь `ready_map`, где ключами будут ID игроков, а значениями — `True/False`.

**Прихо логика:**
1. Игрок нажимает кнопку (например, `Space`).
2. Клиент меняет свой локальный статус и отправляет событие `"ready"` всем остальным.
3. Все получатели обновляют свой `ready_map`.

## 2. Авторитет Хоста

В мультиплеере часто один из участников (Хост) имеет больше прав. В нашем случае:
- Только Хост может отправить событие `"start"`.
- Хост проверяет, все ли игроки в `ready_map` имеют статус `True`.

**Пример проверки на Хосте:**
```python
# Если я — Хост и все готовы
if ctx.is_host and all(ready_map.values()):
    ctx.send("start", {})
    # Переходим в режим игры
    game_started = True
```

## 3. Обработка входящих событий готовности

Важно правильно обновлять состояние при получении сообщения от другого игрока.

**Пример кода:**
```python
for msg in ctx.poll():
    event = msg.get("event")
    data = msg.get("data")
    
    if event == "ready":
        player_id = data["id"]
        is_ready = data["value"]
        ready_map[player_id] = is_ready
        
    elif event == "start":
        # Это событие пришло от Хоста
        game_started = True
```

## Практика

1. Изучите `example_ready_state.py`.
2. Запустите два окна. Нажмите `Space` в одном — вы увидите, как статус обновился в обоих окнах.
3. Когда оба игрока станут `Ready`, появится надпись `Start`, и через секунду игра «начнется» (изменится надпись State).
