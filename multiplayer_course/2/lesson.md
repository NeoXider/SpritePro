# Урок 2. Синхронизация позиций и состояний

## Цель

Научиться передавать координаты игроков между клиентами, использовать троттлинг для оптимизации трафика и правильно разделять «своего» и «чужого» игрока.

## 1. Состояние vs События

В сетевых играх данные делятся на два типа:

| Тип | Частота | Примеры | Как отправлять |
|-----|---------|---------|---------------|
| **Событие** | Однократно | «Выстрел», «Открыта дверь», «Готов» | `ctx.send("shoot", ...)` |
| **Состояние** | Каждый кадр | Позиция, HP, угол поворота | `ctx.send_every("pos", ..., interval)` |

Позиция — это состояние. Мы **регулярно** сообщаем всем, где находимся.

## 2. MultiplayerContext (`ctx`)

В SpritePro при запуске с `multiplayer=True` создаётся объект контекста. Он инкапсулирует отправку, приём и роль:

```python
ctx = s.multiplayer_ctx
```

| Свойство / метод | Описание |
|-----------------|----------|
| `ctx.is_host` | `True` если этот процесс — хост (сервер + игрок 1) |
| `ctx.client_id` | Уникальный ID этого игрока (int) |
| `ctx.id_assigned` | `True` когда ID получен от сервера |
| `ctx.send(event, data)` | Отправить сообщение |
| `ctx.poll()` | Получить список входящих `[{event, data, sender_id}, ...]` |
| `ctx.send_every(event, data, interval)` | Отправить не чаще чем раз в `interval` секунд |

## 3. Троттлинг (Ограничение частоты)

При 60 FPS и отправке пакета каждый кадр — это 60 пакетов/сек. Для пинг-понг позиции достаточно 10-20 пакетов/сек.

**`send_every` решает эту проблему:**

```python
# Отправляем позицию не чаще чем 10 раз в секунду
pos = me.get_world_position()
ctx.send_every("pos", {"pos": list(pos)}, 0.1)
```

Под капотом `send_every` запоминает время последней отправки и отправляет пакет только если прошло >= `interval` секунд.

> **Почему `list(pos)`?** `pos` — это `pygame.Vector2`, который нельзя сериализовать в JSON. `list(pos)` → `[100.0, 200.0]`.

## 4. Обработка «Чужих» игроков

На каждом клиенте есть:
- **«Свой» игрок** (`me`) — управляется клавиатурой.
- **«Чужой» игрок** (`other`) — управляется данными из сети.

```
Клиент 1 (Хост)              Клиент 2
┌─────────────────┐          ┌─────────────────┐
│ me (WASD)       │──pos──→  │ other (из сети)  │
│ other (из сети) │←──pos──  │ me (WASD)        │
└─────────────────┘          └─────────────────┘
```

## 5. Полный пример: синхронизация двух игроков

```python
"""Синхронизация позиций двух игроков."""
import pygame
import spritePro as s


def multiplayer_main():
    s.get_screen((800, 600), "Lesson 2 - Sync Positions")
    ctx = s.multiplayer_ctx

    # --- Создаём два спрайта: «я» и «другой» ---
    me = s.Sprite("", (40, 40), (200, 300))     # Свой
    other = s.Sprite("", (40, 40), (600, 300))   # Чужой

    # Цвета по роли: хост — красный, клиент — синий
    if ctx.is_host:
        me.set_color((220, 70, 70))       # Красный
        other.set_color((70, 120, 220))    # Синий
    else:
        me.set_color((70, 120, 220))       # Синий
        other.set_color((220, 70, 70))     # Красный

    # Буфер для позиции другого игрока (обновляется из сети)
    remote_pos = list(other.get_world_position())
    speed = 240.0

    # --- Игровой цикл ---
    while True:
        s.update(fill_color=(20, 20, 25))
        dt = s.dt  # Время кадра (секунды)

        # 1. Управление: WASD / стрелки
        dx = s.input.get_axis(pygame.K_a, pygame.K_d)  # -1, 0, или +1
        dy = s.input.get_axis(pygame.K_w, pygame.K_s)
        pos = me.get_world_position()
        pos.x += dx * speed * dt
        pos.y += dy * speed * dt
        me.set_position(pos)

        # 2. Отправляем позицию (не чаще 5 раз/сек)
        ctx.send_every("pos", {"pos": list(pos)}, 0.2)

        # 3. Читаем входящие и обновляем чужого
        for m in ctx.poll():
            if m.get("event") == "pos":
                data = m.get("data", {})
                remote_pos[:] = data.get("pos", [0, 0])

        # 4. Применяем позицию чужого
        other.set_position(remote_pos)


if __name__ == "__main__":
    # Запуск: создаёт сервер + 2 клиента автоматически
    s.run(multiplayer=True, multiplayer_entry=multiplayer_main)
```

**Разбор ключевых строк:**

| Строка | Что делает |
|--------|-----------|
| `s.input.get_axis(K_a, K_d)` | Возвращает `-1` (A), `0` (ничего) или `+1` (D) |
| `dx * speed * dt` | Скорость * время кадра = плавное движение независимо от FPS |
| `ctx.send_every("pos", ..., 0.2)` | Отправка позиции максимум 5 раз/сек |
| `remote_pos[:] = ...` | Обновляем список **на месте** (не создаём новый объект) |
| `other.set_position(remote_pos)` | Телепортируем чужого спрайта в последнюю известную позицию |

## 6. Что происходит при запуске

1. `s.run(multiplayer=True, ...)` поднимает TCP-сервер (хост) и второй процесс (клиент).
2. Каждый процесс вызывает `multiplayer_main()`.
3. Хост видит красный квадрат (свой) и синий (чужой).
4. Клиент видит синий (свой) и красный (чужой).
5. При движении WASD позиция отправляется через `send_every`.
6. Другой процесс получает её через `poll()` и двигает свой `other`.
