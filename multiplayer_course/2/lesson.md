# Урок 2. Синхронизация позиций и состояний

В этом уроке мы научимся передавать координаты игроков между клиентами и разберем, почему «наивный» подход к сети может положить сервер.

## 1. Состояние vs События

В сетевых играх данные делятся на два типа:
- **События (Events)**: Разовые действия. *«Игрок выстрелил»*, *«Дверь открылась»*.
- **Состояние (State)**: Постоянно меняющиеся данные. *«Позиция игрока»*, *«Здоровье»*, *«Угол поворота»*.

Позиция — это состояние. Мы должны регулярно сообщать всем остальным, где мы находимся.

## 2. Троттлинг (Ограничение частоты)

Если ваша игра работает при 60 FPS и вы отправляете пакет каждый кадр, сеть быстро забьется. Для плавного движения в простых играх достаточно отправлять позицию 10-20 раз в секунду.

В SpritePro для этого есть метод `ctx.send_every(event, data, interval)`. Он отправит сообщение только если с момента последней отправки прошло `interval` секунд.

**Пример эффективной отправки:**
```python
# Отправляем позицию не чаще, чем 10 раз в секунду (0.1 сек)
pos = me.get_world_position()
ctx.send_every("pos", {"x": pos.x, "y": pos.y}, 0.1)
```

## 3. Обработка «Чужих» игроков

На каждом клиенте есть «свой» игрок (которым мы управляем) и «чужие» (которыми управляет сеть).

**Логика обновления:**
1. Опрашиваем `ctx.poll()`.
2. Если пришло сообщение `"pos"`, обновляем координаты соответствующего спрайта.

**Пример кода:**
```python
for msg in ctx.poll():
    if msg.get("event") == "pos":
        data = msg.get("data")
        # Обновляем координаты другого игрока
        other_player.set_position((data["x"], data["y"]))
```
